buildme

Zero-config container image builds with semantic versioning, registry auto-detection, and post-build hooks.

Docker + Podman Auto Versioning Registry Detection SELinux Aware Post-Build Hooks Self-Update Multi-Service Dry-Run Mode

Set variables for your buildme.conf. Unconfigured values are included as commented lines.

      

    


    
    

      
      
    

  






  
How it works

  

    

      
01

      
Drop it in

      
Place buildme.sh at the root of any repo containing a Dockerfile.

    

    

      
02

      
Run it

      
Execute ./buildme.sh — it auto-detects your registry, engine, and build mode.

    

    

      
03

      
Version & Push

      
Semantic versioning (major.yyddd.incremental) is applied and images are pushed automatically.

    

    

      
04

      
Extend

      
Add hooks in buildme.d/ or use --extensions to grab community hooks.
